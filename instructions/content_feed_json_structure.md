# Content Feed JSON Structure

This document describes the JSON structure currently generated by:

- `feed_module.content.generate_content_xml(...)`

The function name still contains `xml`, but the current implementation writes JSON.

## Purpose

This feed represents product content data for items that are eligible for export.

Current public endpoint:

- `GET /api/content-feed.xml`

Important:

- the URL still ends with `.xml`
- the response body is JSON

## Top-Level JSON Shape

The function always writes a JSON object with one top-level key:

```json
{
  "offers": []
}
```

- `offers`: array of product content objects

The file is written as pretty-printed UTF-8 JSON with indentation.

## Inclusion Rules

An offer is included only when all of the following are true:

1. The source offer is available.
2. The source quantity is greater than zero.
3. A barcode can be resolved.

Current barcode resolution order:

1. `override.barcode`
2. source offer `<barcode>`
3. supplemental source offer `<barcode>` matched by the same `offer id`

Important:

- numeric `Артикул` values are not treated as barcodes
- supplemental barcode lookup is only done by the same `offer id`
- supplemental source may still be used for images by `offer id`, with `vendor_code` fallback for images only

## Offer Object Structure

Each item in `offers[]` is a JSON object with these keys, in this current order:

1. `id`
2. `code`
3. `vendor_code`
4. `title`
5. `barcode`
6. `category`
7. `category_id`
8. `brand`
9. `availability`
10. `weight`
11. `height`
12. `width`
13. `length`
14. `description`
15. `image_link`
16. `tags`

All of these keys are always present in every exported item.

## Field Reference

| Key | Type | Nullable / Empty | Source / Logic | Notes |
|---|---|---:|---|---|
| `id` | `string` | No | `source_offer.offer_id` | Source system item id. |
| `code` | `string` | No in normal flow | `override.code` or `source_offer.offer_id` | Used to match the propositions feed. |
| `vendor_code` | `string \| null` | Yes | `source_offer.vendor_code`, otherwise supplemental `vendor_code` from image-enrichment match | May be `null` if absent in both sources. |
| `title` | `string` | No | Normalized source title | Trimmed and normalized by `normalize_title(...)`. Current logic limits it to 100 characters. |
| `barcode` | `string` | No for exported items | Resolved barcode | Export filter depends on this. Items without barcode are skipped completely. |
| `category` | `string \| null` | Yes | `override.category` or source leaf category name | Current source may leave this `null`. |
| `category_id` | `string \| null` | Yes | `override.category_id` or source `categoryId` | Current source may leave this `null`. |
| `brand` | `string \| null` | Yes | `override.brand`, source brand, extracted brand, or inferred brand | Brand inference is implemented. |
| `availability` | `string` | No | Constant | Always `"Є в наявності"` for exported items. |
| `weight` | `string \| null` | Yes | `override.weight_kg` or parsed source measure | Stored as text, not numeric JSON. |
| `height` | `string \| null` | Yes | `override.height_cm` or parsed source measure | Stored as text. |
| `width` | `string \| null` | Yes | `override.width_cm` or parsed source measure | Stored as text. |
| `length` | `string \| null` | Yes | `override.length_cm` or parsed source measure | Stored as text. |
| `description` | `string \| null` | Yes | Sanitized source description, enriched by title params, or generated from tags | HTML string, not structured JSON. |
| `image_link` | `array[string]` | Yes, can be `[]` | Deduplicated merge of override images, main source images, supplemental images | Current function keeps this key even when the array is empty. |
| `tags` | `array[object]` | Yes, can be `[]` | Merged source params, description-extracted params, title-extracted params, and override replacements | Keeps repeated names as separate entries when needed. |

## `description` Field

`description` is a string containing HTML.

Current generator behavior:

- sanitizes HTML to the supported subset
- removes forbidden sections such as delivery, payment, warranty, contacts, and configuration
- may append parameters derived from the title
- if the source description is empty, generates a description from the resolved tags

Current allowed tag set in the generated description logic:

- `<h5>`
- `<p>`
- `<ul>`
- `<li>`
- `<br>`
- `<img alt="" src="...">`

`description` can be `null` if the generator still cannot produce content.

## `image_link` Field

`image_link` is a JSON array of URLs.

Example:

```json
"image_link": [
  "https://example.com/image-1.jpg",
  "https://example.com/image-2.jpg"
]
```

Rules in the current implementation:

- images are deduplicated
- empty or blank URLs are discarded
- order is preserved by source priority:
  1. override images
  2. main source images
  3. supplemental images

## `tags` Field

`tags` is a JSON array of objects in this structure:

```json
{
  "name": "Матеріал",
  "value": "боросилікатне скло"
}
```

Important:

- `tags` is never a dictionary/map
- duplicate `name` values are allowed as separate array entries
- this is how multiselect parameters are preserved

Current tag-building sources:

1. original source params
2. params extracted from the HTML description
3. params extracted from the normalized title
4. override params, which replace same-name existing params
5. fallback synthetic tag `{"name": "Тип", "value": ...}` when no tags are found

## Implementation Notes

- The current function writes JSON to disk even though its name is `generate_content_xml`.
- The function may still export `null` for required business fields such as `category`, `category_id`, `weight`, `height`, `width`, and `length` when source data is missing and no override is provided.
- If `strict=True`, the function validates required business fields after building the payload and raises an error if any exported item is incomplete.
- The JSON structure itself remains stable even when values are `null` or arrays are empty.
